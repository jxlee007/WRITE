# Unified Token & Media System Migration Guide

## Overview

The token and media library systems have been merged into a single unified system. All images (both uploaded and AI-generated) are now stored as tokens with specific type designations.

## Key Changes

### Schema Changes

The `tokens` table now includes all media-related fields:
- `source`: "uploaded" or "ai-generated"
- `fileUrl`: Storage URL or Convex storage ID
- `thumbnailUrl`: Optional preview thumbnail
- `fileName`: File name
- `fileSize`: Size in bytes
- `mimeType`: MIME type
- `dimensions`: `{ width: number, height: number }`
- `prompt`: AI generation prompt (for AI-generated images)
- `modelUsed`: AI model name (for AI-generated images)
- `settings`: AI generation settings (for AI-generated images)

### New Token Types

1. **`reference-image`**: Any image uploaded anywhere in the app
   - Automatically created when images are uploaded
   - Stored with full metadata (dimensions, file size, etc.)

2. **`ai-generated-image`**: Any image generated by AI
   - Automatically created when AI generates images
   - Includes prompt, model, and generation settings

3. **Traditional tokens**: `character`, `location`, `object`, `creature`, `faction`, `event`
   - Unchanged, can still have optional images attached

### New Unified APIs

#### For Uploaded Images (Reference Images)
```typescript
// âś… NEW - Use these
api.tokens.uploadReferenceImage({
  projectId,
  name: "My Reference Image",
  description: "Description",
  fileUrl: storageId,
  fileName: "image.png",
  fileSize: 123456,
  mimeType: "image/png",
  dimensions: { width: 1024, height: 768 }
})

// Get all reference images
api.tokens.getTokens({ 
  projectId, 
  type: "reference-image" 
})
```

#### For AI-Generated Images
```typescript
// âś… NEW - Use these
api.tokens.saveAIGeneratedImage({
  projectId,
  name: "AI Generated - Character Portrait",
  description: "Optional description",
  prompt: "Full AI generation prompt",
  fileUrl: "https://...",
  modelUsed: "stable-diffusion-xl",
  settings: {
    size: "1024x1024",
    quality: "hd",
    style: "vivid"
  },
  dimensions: { width: 1024, height: 1024 }
})

// Get recent AI-generated images
api.tokens.getRecentAIImages({ 
  projectId, 
  limit: 10 
})
```

#### For All Media Tokens
```typescript
// Get both reference and AI-generated images
api.tokens.getMediaTokens({ 
  projectId, 
  type: "all" // or "reference-image" or "ai-generated-image"
})

// Delete any token (including images)
api.tokens.deleteToken({ id: tokenId })

// Generate upload URL (moved to tokens)
api.tokens.generateUploadUrl()
```

## Component Updates

### TokenLibrary Component
- Now displays all token types including image tokens
- Upload button automatically creates `reference-image` tokens
- Visual indicators for reference vs AI-generated images
- Filter by token type (characters, locations, images, etc.)

### MediaLibrary Component
- Now uses unified token system
- Shows both reference and AI-generated images
- Upload creates `reference-image` tokens automatically
- Note: Video/audio support coming in future update

### AIGenerator Component
- Saves generations as `ai-generated-image` tokens
- Includes full prompt and generation metadata

### RecentGenerationsSidebar
- Uses `api.tokens.getRecentAIImages()`
- Compatible with new token structure

## Automatic Image Storage

### Upload Anywhere â†’ Reference Image
Any image uploaded anywhere in the application is automatically stored as a `reference-image` token:
- Image uploads in TokenLibrary
- Image uploads in MediaLibrary
- Future: Image uploads in editor, chat, etc.

### AI Generate â†’ AI Generated Image
Any image generated by AI is automatically stored as an `ai-generated-image` token:
- Images from AIGenerator
- Future: Images from chat AI, inline generation, etc.

## Benefits

1. **Unified System**: All content in one place - no separate media and token systems
2. **Better Organization**: Images are first-class tokens with full metadata
3. **Consistent API**: Single set of APIs for all token operations
4. **Rich Metadata**: Full prompt, model, and settings tracking for AI images
5. **Flexible Queries**: Filter by type, source, date, etc.
6. **Future-Proof**: Easy to add video/audio as token types later
